# Verification Status

Open-source formal flow: OSS CAD Suite (**yosys 0.66 + SymbiYosys + boolector**).
Run everything: `cd formal && ./run.sh`  (or `./run.sh <name>` for one module).
Tasks per module: **bmc** (bounded bug-hunt), **prove** (k-induction, *unbounded*),
**cover** (reachability — a PASS means every cover point is reachable, so the safety
asserts are non-vacuous).

**Full suite: ALL GREEN** — 14 modules + integration, **41 proof tasks**, **~280
assertions**. Independently reproduced from scratch (workdirs wiped, paths
standardized) and **re-confirmed green after every post-sim RTL fix** (5-bit register
index, DAA `bit_resync`, FIFO `clear`, front-end release-tail `OE_TAIL`, bit-engine
`tx_first`, FSM `read_done_q`, live-`rnw` CCC-ACK decode).

The combinational Avalon **output**-pin paths (avs_readdata/waitrequest/irq) are pad-buffer-limited in standalone pin synthesis (-2.86 ns) -- an on-chip-IP-boundary / OOC artifact, NOT an in-system path; see syn/altera/README.md. 528 ALMs / 348 regs / 2 RAM blocks |

The three checks are complementary: formal proves the per-module state machines
exhaustively on idealized one-cycle edge strobes; simulation exercises real end-to-end
transactions over an oversampled open-drain bus (which is what exposed the integration
timing bugs below); Quartus proves the device-agnostic RTL maps to real silicon and
closes timing with comfortable margin (positive setup **and** hold slack at the 8.000 ns
target).

Simulation coverage (all PASS): broadcast `0x7E+W` ACK; **full ENTDAA** (64-bit payload,
assigned DA latches as `0x08`, target ACKs the address); **private write** (`0x5C` into
the RX FIFO, read back over Avalon); **private read** (target byte `0xC3`); **GETSTATUS**
(7E+W ACK, DA+R ACK, single-byte status response); plus the Avalon register surface
(PID/BCR/DCR identity, GETCAPS `0x0200`, CTRL R/W, DYN_ADDR, TX-FIFO push/level,
`flush_tx`).

## Bugs found by simulation and fixed (7) + 1 tracked

Simulation found **8 integration bugs the per-module formal proofs could not** — formal
uses idealized one-cycle edge strobes, so only real oversampled bus timing exposes
release-tail, byte-framing, and read-termination effects. **7 are fixed and re-verified**
(full formal suite still ALL GREEN, Quartus build still clean); **1 is tracked open**.
Full write-ups in `docs/findings.md`.

| # | Bug | Fix (RTL signal / module) | State |
|---|---|---|---|
| 1 | **GETCAPS/RESET read = 0** — 4-bit `app_*_idx` aliased register indices 16/17 to 0 | widened `app_wr_idx`/`app_rd_idx` to **5-bit** (`i3c_regfile` + `i3c_avalon_mm` + top) + CAPS/RESET read decode | ✅ fixed |
| 2 | **FINDING-SIM-1** — driven SDA slot released ~2 cyc after the *synchronized* 9th rising while synced-SCL still High → front-end sees SDA-rise-while-SCL-high = false STOP → frame collapses | front-end **release tail** `OE_TAIL` (default 4) in `i3c_bus_frontend`: bus-condition detection stays gated a few cycles after `sda_oe` deasserts | ✅ fixed |
| 3 | **FINDING-SIM-2** — `flush_tx`/`flush_rx` generated by Avalon but unwired | synchronous **`clear`** port added to `i3c_fifo`, wired to the flush pulses in top | ✅ fixed |
| 4 | **FINDING-SIM-3** — DAA assigned-address byte misframed (64-bit continuous payload, `64 mod 9 ≠ 0`, so the shared bit engine's 9-bit framing drifted) | **`bit_resync`** input on `i3c_bit_engine`, pulsed by `i3c_daa` (`rxda_enter`) when entering the DA-receive state | ✅ fixed |

| 7 | **FINDING-SIM-6** — every directed GET (GETSTATUS/GETPID/…) wrongly NACKed (`is_read` used the *latched* `rnw_q`, stale at the CCC-ACK decision on `byte_done`) | `i3c_protocol_fsm` drives `is_read = ` **live `rnw`** at `S_ADDR && byte_done`, else `rnw_q` | ✅ fixed |

## Known gaps (honest accounting — to close before tape-out)

1. **Integration F-1/F-2/F-3 are BMC-bounded (depth 40) + cover (depth 120), NOT
   k-induction.** basecase + BMC pass; the unbounded induction step needs auxiliary
   reachable-state invariants (notably `a_f3_top`, the registered ADAPT-1 gate — the F-7
   risk from `docs/critique.md`). The integration `.sby` deliberately omits a `prove`
   task and strips every per-module `$check` so its results never rely on unit assumes.
   **Per-module properties *are* unbounded (k-induction).** Top hardening task: add the
   invariants that make the cross-cutting top properties inductive.
2. **FINDING-SIM-7 (open) — multi-byte GET responses drive only the first byte.**
   `resp_idx` in `i3c_ccc` advances at `byte_done`, so the byte-0 T-bit already sees
   `resp_idx==1` and ends the response early. Single-byte GETs (e.g. GETSTATUS as run in
   sim), the directed-GET ACK, and response-start all work; a decoupled response pipeline
   is needed for ≥2-byte GETs (GETPID, GETBCR/GETDCR chains, GETMWL/GETMRL). Tracked in
   `docs/findings.md`.
3. **Deferred / owned-elsewhere properties** (documented, not yet asserted): deep read
   T-bit + MRL-bound end-to-end (T2/T3/T6 across framer+FSM+integration); TE6 read-back
   (optional, needs line read-back); IBI PRN + MDB-format legality (I10/I11); private-read
   double-NACK collision outcome (IBI-DEFER-01 case b); RXF_LAST frame-boundary marker;
   MWL write-overflow abort (relies on the FIFO overflow flag today).
4. **Assume↔assert ledger (F-9):** every standalone-proof `assume` must be discharged by
   a neighbor's proven `assert`. Tracked in `docs/assume_ledger.md`; the integration proof
   already strips per-module checks so its results don't rely on unit assumes. A CI script
   to auto-verify every ledger entry is still pending.
5. **Board bring-up not done (device-agnostic by design).** The Quartus build closes
   timing for Cyclone 10 GX but has **no pin-location assignments** (`set_location_assignment`
   for SDA/SCL/clk/Avalon) and uses the **inferred tri-state** SDA pad. A real board needs
   pin locations and, for the open-drain pad, the bus IO standard and likely an **ALTIOBUF**
   instance swapped into `rtl/altera/i3c_io_altera.sv`. The deliberately-cut async SDA/SCL
   paths (`set_false_path` in the SDC) are closed by the 2-FF synchronizers + simulation.
6. **Product decisions still open** (parameterized placeholders, see `docs/design_decisions.md`
   §6): MIPI **Manufacturer ID + PID** (PartID/InstanceID, PID type); **MWL/MRL/MaxIBIPayload**
   reset defaults & maxima and out-of-range policy; **GETCAP3 Format-2 bit index** (prose
   bit4 vs Table 63 bit3); `tSCO ≤ 20 ns` gate (flips BCR[0]/GETMXDS if the path can't meet it).
